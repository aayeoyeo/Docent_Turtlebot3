## 1. 서브시스템별 단위 테스트 결과표

본 표는 각 서브시스템별로 필수 기능을 검증한 단위 테스트 결과입니다. 요구사항 중 U-10과 U-11은 현재 시스템 환경적 한계로 인해 부분 구현 및 우회 구현 상태로 관리되고 있습니다.

| ID | 대상 | 테스트 내용 | 기대 결과 | 실제 결과 | 판정 | 담당 |
|---|---|---|---|---|---|---|
| U-01 | AI / 통신 | 스마트폰 카메라에 전시물 QR 코드 E1 노출 | /exhibit_seen 토픽으로 "E1" 문자열이 즉시 발행됨 | 즉시 발행 확인 및 정지 연계 성공 | ✅ | 최준혁 |
| U-02 | AI / 통신 | tracking.js 엔진 전방에 관람객 안면 인식 | /audience 토픽으로 true 플래그가 연속 발행됨 | 안면 매핑 및 토픽 발행 확인 | ✅ | 최준혁 |
| U-03 | AI / 통신 | /tts_text 토픽으로 해설 스크립트 수신 | Web Speech API 가동 및 한국어 음성 정상 출력 | 폰 스피커로 음성 출력 완료 | ✅ | 최준혁 |
| U-04 | ROS 노드 | /exhibit_seen으로 현재 타겟 ID (E1) 수신 | 로봇 Phase 상태가 MOVE에서 EXPLAIN으로 즉시 전이됨 | Phase 시퀀스 전환 확인 | ✅ | 여태인 |
| U-05 | ROS 노드 | EXPLAIN 상태에서 /tts_done (true) 수신 | 로봇 Phase 상태가 CHECK로 전이되며 타이머 카운트 시작 | 상태 전이 및 시간 누적 시작 | ✅ | 여태인 |
| U-06 | ROS 노드 | CHECK 상태에서 20초간 관람객 안면 미감지 | 관람객 부재 타임아웃 예외 처리 가동, 차기 인덱스로 강제 이동 | 20초 후 자동 주행 복귀 확인 | ✅ | 여태인 |
| U-07 | 웹 프런트 | 코스 편집기에서 전시물 순서 변경 후 [적용] 클릭 | 변경된 순서 구조의 JSON 데이터가 /course 토픽으로 엣지 발행됨 | JSON 패킷 발행 및 로봇 수신 완료 | ✅ | 김대권 |
| U-08 | 웹 프런트 | 로봇으로부터 /tour_state JSON 패킷 수신 | 관리자 대시보드 화면의 진행률, 현재 전시물 및 Phase 텍스트 동기화 | 대시보드 UI 실시간 갱신 확인 | ✅ | 김대권 |
| U-09 | 웹 프런트 | 스마트폰 뷰어앱 브라우저를 백그라운드로 전환 | visibilitychange 이벤트 감지 즉시 로봇 정지 명령 발행 | 감속 및 /cmd_vel 제로화 확인 | ✅ | 김대권 |
| U-10 | HW / 화면 | 로봇 중앙 노드로부터 차기 타겟 전시물 데이터 수신 | 물리 LCD 대체의 건으로 스마트폰 UI 상단 레이어에 해당 전시물 제목이 오버레이 출력됨 | 스마트폰 화면 상단 디스플레이 우회 표시 확인 | ⚠️ 부분 | 여태인 |
| U-11 | 콘텐츠 | 웹앱 설정에서 TTS 재생 속도 및 다국어 보이스 파라미터 제어 | 설정된 배속(0.8~1.2) 제어 반영 및 출력 | 음성 변경을 제외한 재생 속도 조절만 작동 완료 | ⚠️ 부분 | 여태인 |
| U-12 | 관람 통계 | 로봇이 각 전시물 EXPLAIN 및 CHECK 상태 체류 | 전시물별 진입/이탈 시간 차를 계산하여 타임스탬프 데이터 실시간 누적 | 전시물별 누적 체류 시간 수집 성공 | ✅ | 여태인 |

## 2. 통합 테스트 결과서

본 섹션은 스마트폰 앱, 중앙 ROS 노드, 관리자 웹앱 및 터틀봇3 하드웨어를 모두 연동한 상태에서 실제 구현된 핵심 기능들을 시나리오 기반으로 검증한 결과(E2E 시나리오)입니다.

| ID | 대상 | 테스트 내용 | 기대 결과 | 실제 결과 | 판정 | 담당 |
|---|---|---|---|---|---|---|
| I-01 | E2E 통합 | 코스 시작 후 전체 전시물(E1➔E2➔E3) 순차 자율 주행 및 인식 테스트 | 각 목적지 QR 정지 오차 ±20cm 이내 안착, 폰 화면 전시물 제목 오버레이 표출 및 해설 20초 이상 송출 후 안면 인식 연계 주행 성공 | 전체 코스 무인 정상 순회 완료 및 오차 범위 내 정지 확인 | ✅ | 여태인 |
| I-02 | E2E 통합 | CHECK 단계에서 의도적인 관람객 전원 이탈 시나리오 수행 | 안면 미인식 상태 지속 20초 경과 시, 예외 처리 정책에 따라 대기 시퀀스를 종료하고 차기 코스로 강제 주행 재개 | 20초 대기 정책 타이머 정상 가동 및 자동 MOVE 전이 확인 | ✅ | 여태인 |
| I-03 | E2E 통합 | 사전 코스 편집 및 변경 데이터 로봇 반영 주행 테스트 | 대기 상태에서 관리자 웹앱을 통해 전시물 순서를 편집한 후 로봇에 주입하여 변경된 경로대로 투어가 시작되는지 검증 | 신규 코스 JSON 데이터 동적 로딩 및 바뀐 순서대로의 정상 자율 순회 확인 | ✅ | 여태인 |
| I-04 | E2E 통합 | 실시간 투어 현황 관제 및 대시보드 데이터 동기화 테스트 | 로봇이 코스를 순회하며 제어 단계 변화가 일어날 때 대시보드 화면에 정보가 실시간 표출되는지 검증 | 제어 단계 전이에 따른 진행률 및 현재 작품 정보 텍스트의 실시간 UI 동기화 확인 | ✅ | 여태인 |
| I-05 | E2E 통합 | 주행 중 관리자 관제탑 레이아웃에서의 수동 모드 강제 개입 조종 | 자율 주행 상태에서 수동 모드 강제 전환 시 자동 제어가 차단되고 D-Pad 제어 명령으로 실시간 원격 이동이 가능한지 검증 | 제어권 회수 플래그 즉시 작동 및 D-Pad 마우스 인터랙션을 통한 실시간 원격 제어 성공 | ✅ | 여태인 |

## 3. 버그 트래킹 표

본 시스템 통합 과정에서 발견되어 조치 완료되었거나 추적 중인 소프트웨어 및 하드웨어 연동 관련 버그 현황입니다.

| ID | 증상 | 원인 | 조치 | 상태 | 담당 |
|---|---|---|---|---|---|
| B-01 | 주행 속도로 인해 로봇이 전시물 QR 코드를 인식하지 못하고 지나치는 현상 | 코스 순서가 고정되어 있어 특정 ID 미인식 시 무한 전진함 | docent_node.py 내에 구간별 이동 최대 타임아웃(30초) 감지 로직을 추가하여 미인식 시 자동 IDLE 정지 유도 | 해결 | 여태인 |
| B-02 | 투어 도중 웹앱에서 신규 코스 수신 시 로봇 시퀀스가 초기화되는 현상 | /course 토픽 수신 시 내부 인덱스를 무조건 0으로 리셋함 | 상태 제어 플래그를 검사하여 IDLE 상태가 아닐 경우 버퍼에 새 코스를 대기시킨 후 투어 종료 시점에 갱신하도록 구조 개선 | 해결 | 여태인 |
| B-03 | 스마트폰 화면이 절전 모드로 진입 시 안면 인식 및 스트리밍이 전면 중단되는 현상 | 모바일 웹 브라우저의 백그라운드 자원 제한 정책에 의한 추론 스레드 차단 | 모바일 스크립트 상단에 Wake Lock API를 연동하여 강제 화면 켜짐 유지 처리 및 관제탑 경고 알림 연동 | 해결 | 여태인 |
| B-04 | 웹앱 TTS 해설문 재생 중 특정 긴 문자열 유입 시 완료 이벤트(/tts_done)가 발행되지 않는 현상 | 브라우저 내 SpeechSynthesisUtterance 객체의 텍스트 길이 버퍼 오버플로우로 인한 비동기 onend 바인딩 누락 | 해설 스크립트 문자열을 문장 단위 기준으로 파싱하여 큐 구조로 분할 연속 재생하도록 웹 로직 보완 | 해결 | 여태인 |
