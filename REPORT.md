# 🤖 자율주행 AI 도슨트 로봇 시스템 최종 보고서 — 4조

## 1. 프로젝트 요약
본 프로젝트는 ROS 2(Humble) 기반의 터틀봇3와 스마트폰 센서를 웹소켓(`rosbridge`)으로 연동하여, 전시관 내에서 관람객을 안내하고 다국어 작품 해설을 제공하는 자율주행 지능형 상태 머신(AI Docent) 시스템을 성공적으로 구축했습니다.
* **계획 대비 달성률 (총합 90% 이상):** * Must(필수) 요구사항 100% 달성 (전시물 순회, QR 인식, 해설, 안면 인식 기반 타임아웃, 예외 처리, 안전장치 등)
  * Should(우선) 요구사항 약 80% 달성 (전시물 체류 통계 완벽 구현, LCD 및 TTS 튜닝은 하드웨어/OS 환경적 한계로 우회 및 부분 구현)

## 4. 핵심 구현 설명 

* **여태인 (메인 코어 개발, 기획, 문서 총괄 및 QA 공동 수행):** 프로젝트의 중추가 되는 ROS 2 메인 상태 머신(`docent_node.py`) 아키텍처 전체 설계 및 코딩, 스마트폰 AI 파이프라인(QR/안면인식) 흐름 제어, 웹소켓 연동 백엔드 로직 작성을 전담했습니다. 전체 필수 산출물(`PLAN.md`, `TEST.md`, `REPORT.md` 등) 작성을 총괄하였으며, 김대권 조장과 함께 시스템 E2E 통합 테스트 및 QA를 공동으로 수행하며 발생한 버그를 직접 디버깅하여 완성도를 보증했습니다.
* **김대권 (조장 / 웹 프런트 및 통합 QA 총괄):** 팀장으로서 전체 프로젝트 일정을 조율하고 시스템 통합(E2E) QA 및 테스트 검토를 총괄했습니다. 여태인 팀원과 협력하여 버그 트래킹을 수행했으며, 관리자 관제탑 대시보드의 기초 UI 마크업 및 프런트엔드 레이아웃 구성을 담당했습니다.
* **최준혁 (모바일 환경 테스트 보조):** `robot.html` 로딩 시 모바일 브라우저 권한 활성화 및 Web Speech API의 기기별 오디오 세션 테스트를 보조했습니다.
* **진승민 (하드웨어 보조):** 주행 테스트 시 물리적 장애물을 제거하고 터틀봇3 배선을 점검했습니다.
* **박현상 (콘텐츠 보조):** 전시물 해설 스크립트 JSON 데이터의 타이핑 및 오타 검수를 보조했습니다.

## 5. 테스트 결과 요약

* **단위 테스트 (12항목):** 핵심 로직 10항목 100% 정상 가동 확인. 하드웨어/OS 종속적인 2항목(LCD 출력, 다국어 TTS 음성팩)은 스마트폰 UI 오버레이 및 배속 조절 기능으로 우회하여 부분 구현함.
* **통합(E2E) 테스트 (5시나리오):** 전 항목(자율 순회, 20초 부재 타임아웃, 투어 중 코스 편집 후 차기 반영, 실시간 현황 관제, 긴급 수동 원격 조종) 무결점 통과 완료.
* **잔존 버그 목록:** 현재 시스템 구동 상의 치명적인 버그는 모두 수정(Zero-Bug)되었으나, 모바일 브라우저 특성상 화면이 완전히 꺼지면 카메라 권한이 일시 중단되는 이슈가 존재하여 Wake Lock API로 화면 꺼짐 방지 조치를 적용한 상태임.

## 6. 트러블슈팅 사례 3선

1. **증상:** 코스 편집기에서 주행 중인 로봇에 신규 코스를 주입할 경우 인덱스가 즉시 0으로 초기화되며 로봇이 오작동함.
* **원인:** `/course` 토픽 구독 콜백에서 상태(Phase)를 고려하지 않고 즉각 리셋을 수행함.
* **해결:** 상태 검사 로직을 추가하여 투어 중(`MOVE`, `EXPLAIN`, `CHECK`)에는 버퍼에 코스를 대기시키고, 투어가 완전히 종료되어 `IDLE` 상태가 되었을 때 갱신되도록 백엔드 코어 제어 흐름 개선.


2. **증상:** 스마트폰 앱을 백그라운드로 전환 시 로봇이 직전 속도를 유지하며 벽에 충돌함.
* **원인:** `/cmd_vel` 워치독이 충족되지 않는 상태에서 연속 명령이 끊어짐.
* **해결:** 웹앱에 `visibilitychange` 이벤트 리스너를 바인딩하여 브라우저 가려짐 감지 시 즉각 `cmd_vel` 속도를 제로(0, 0)로 발행하게 하여 안전장치(M7) 확보.


3. **증상:** 해설 스크립트가 길어질 경우 재생이 완료되었음에도 `/tts_done` 토픽이 발행되지 않음.
* **원인:** 모바일 브라우저 SpeechSynthesis 객체의 긴 문자열 오버플로우로 인해 비동기 `onend` 이벤트가 누락됨.
* **해결:** 백엔드에서 전송할 때 텍스트를 마침표 단위로 분할 큐(Queue) 배열로 만들어 연속 호출하도록 로직 최적화.



## 7. KPT 회고

### 7.1 조 KPT

* **Keep:** 명확한 토픽 인터페이스 정의(`INTERFACE.md`)를 Day 1에 확고히 다져놓고 조장 주도하에 꼼꼼한 QA를 공동 수행한 덕분에 데이터 통신 오류(파싱 에러 등)가 한 건도 발생하지 않은 점을 높이 평가함.
* **Problem:** 로봇의 물리적 하드웨어 한계(LCD 부재)와 폐쇄적인 모바일 OS 정책(오디오 세션)으로 인해 코드로 극복할 수 없는 제약 사항이 많았음. 소프트웨어 코어 개발 및 문서화 업무가 여전히 특정 팀원에게 편중되는 경향이 있었음.
* **Try:** 차기 프로젝트 진행 시, Day 1 단계에서부터 가용 하드웨어 자원을 명확히 테스트하여 불가능한 요구사항을 조기에 제외하고 업무 밸런스를 균형 있게 재분배할 필요가 있음.

### 7.2 개인 KPT

* **여태인:** (Keep) 백엔드 분야를 집중적으로 공부하며 쌓은 아키텍처 설계 역량을 바탕으로 복잡한 비동기 상태 머신과 예외 처리 로직을 안정적으로 완수하고, 조장과 함께 QA를 진행하며 시스템 완성도를 높여 기쁘다. (Problem) 코딩과 전체 산출물 작성을 거의 다 감당하느라 물리적 시간이 부족했다. (Try) 모듈화된 소스 코드 작성법을 도입하여 협업자에게 쉽게 로직을 인수인계하는 방법을 연구해봐야겠다.
* **김대권:** (Keep) 조장으로서 통합 QA를 총괄하며 웹 프론트엔드와 백엔드가 연결되는 전체 시스템의 흐름을 성공적으로 검증할 수 있었다. (Problem) 깊이 있는 백엔드 코어 로직 구현에는 직접 기여하지 못해 아쉽다. (Try) JavaScript와 WebSockets에 대한 기초 역량을 길러 다음에는 통신 로직까지 직접 짜보고 싶다.
* **최준혁:** (Keep) 스마트폰 센서(카메라, 마이크)를 웹 API로 제어하는 개념을 익혔다. (Problem) 모바일 환경에서 발생하는 예기치 못한 브라우저 제약들을 혼자 디버깅하기 어려웠다. (Try) 클라이언트 사이드 디버깅 툴 사용법을 깊이 익혀야겠다.
* **진승민:** (Keep) ROS 2 기반 하드웨어가 실제로 어떻게 결선되고 구동되는지 직접 확인했다. (Problem) 소프트웨어 개발 역량이 부족하여 단순 하드웨어 보조에 머물렀다. (Try) 파이썬 기초와 ROS 2 CLI 명령어부터 차근차근 복습해야겠다.
* **박현상:** (Keep) 사용자가 듣기 편안한 도슨트 해설 대본을 고민하고 기획하는 과정이 흥미로웠다. (Problem) 텍스트 정리나 타이핑 등 보조적인 역할에 그친 점이 못내 아쉽다. (Try) 코딩 역량을 키워 직접 웹 UI 화면에 텍스트를 입히는 작업을 해보고 싶다.
